What is React?

React is a powerful and influential Javascript library developed by Facebook and open sourced a few years ago. It has since gained in popularity throughout the Javascript world, despite some initial skepticism. It describes itself as the 'V' in the MVC model, uses a virtual DOM to render HTML updates in a very efficient manner, and allows you to write HTML within your Javascript code (really!).

Today React comes with a wide range of tools and additional libraries that allow entire applications to be build with React, and there are even projects like React Native that port React to native mobile systems. React works by breaking everything down into components and places strict but clear rules on how data can flow between, and change, these components. This makes designing an application conceptually simple and also breaks the code into discrete, composable pieces that are very easy to work with.

Learning the React Ecosystem

After finishing the Front End Certification at Free Code Camp, I proceeded confidently into the Data Visualization Projects... only to be quickly shut down. Components, ES6, render functions, state? The instructional content for these Free Code Camp challenges was not been completed yet at this time, and it became clear to me that Google searching my way each error message wasn't going to work very well. I didn't even know what a React component was! I needed a strong foundation in React before I could continue.

I decided to start from scratch and build 25 projects using React. I would only proceed to the Free Code Camp challenges when I had a very clear idea of how I would build them. As my knowledge of React grew, I decided to go back and re-build all of the major Front End Projects. This was actually a very interesting experience because it allowed me to see a comparison of how I could build these projects without, and then with, React. I summarized that experience and analyzed the two attempts here.

At this point I had finished about 20 projects using React, and I felt ready to return to the React Challenges in Free Code Camp. I did so, and successfully completed them, working much more efficiently than I otherwise would have been capable of. I think this is largely due to slowing building up my skills from the very basics to more advanced topics before even attempting these challenges.

I was only able to accomplish all of this because there are some smart developers out there who have taken the time to create really excellent quality educational content. To help any future newcomers to the React world, I've compiled below a list of the best resources I came across while learning React (as of June 2016!). Now, everyone has a unique learning style and will be at a different place in their knowledge as a developer when trying to learn something new like React. Maybe you've been building Angular apps for years, or maybe you don't even know what an MVC is (like me!). These resources worked really well for me, but each person may find different resources to be better, and new material will certainly be created in the future (I'm writing this in July 2016). However, I can vouch for the quality of all of the following. These are organized roughly in order of beginning to advanced level material, and illustrate what I think to be a good learning pathway for React JS and everything that comes with it.

1) Thinking in React — A great place to start. All of the Facebook documentation is excellent and a great resource to keep handy.

2) Thinking in React Screencast Tutorial — A very nice and short Youtube tutorial by Hendrik Swanepoel of Tagtree demonstrating many of the 'Thinking in React' principles.

3) React for Everyone — LevelUpTuts Video Tutorial Series introducing you to React. This is a great place to start with the basics.

4) Basic React Tutorial Series — Awesome free video series by Joe Maddalone on Egghead.io walking you through a number of basic React demonstrations. You begin working with webpack and babel in the series, which I recommend, but you can also quickly start writing React in Codepen.

5) 9 things every React.js beginner should know — Some React best practices by full stack developer Cam Jackson.

6) Presentational and Container Components — Dan Abramov on defining components in React.

7) ReactJS Program — Tyler McGinnis has created these excellent React tutorials, the first of which is free. They are more advanced, but I cannot recommend these resources enough.
